Cascade semantics. The matching engine assigns exactly one status per PO line, evaluated in precedence order: missing ASN → missing invoice → UoM mismatch → quantity mismatch → shipped-not-invoiced → missing payment → short pay. A line failing an earlier check is never evaluated against later checks — the first failure is the one that matters for triage. Per-class counts therefore reflect the primary failure mode, not every possible divergence on that line.
